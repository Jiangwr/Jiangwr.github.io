大数据的简单认识

Jan 19, 2019

随着存储能力、CPU 计算能力、网络带宽的增强，也因为物联网的兴起，大数据时代就此来临。

大数据的基本特点(4V)

VOLUME(大量化)、VRLOCITY(快速化)、VARIETY(多样化)、VALUE(低价值)

大数据带来的改变

过去受于技术水平的限制，都是把样本数据抽样出来之后进行分析，靠对样本数据的分析来预测全样数据。

现在是全样分析而不是抽样分析，追求效率，追求数据的相关性。

数据流通的基本过程

数据采集 => 数据存储和管理 => 数据处理与分析

数据采集:
  • ETL工具
  • 爬虫工具

采集之后通过清洗、转换、集成，然后加载到数据仓库

数据存储和管理
  • 分布式文件系统
  • 分布式数据库(包含非关系型数据库)
数据处理与分析

通过分布式计算框架

  • MapReduce
  • Spark

大数据相关的主要技术

大数据主要涉及以下两个技术层面:

  1. 分布式存储:解决数据存储问题
  2. 分布式计算:解决数据计算问题

Hadoop 生态中对应的技术框架

  1. 分布式存储框架:HDFS

  2. 分布式计算框架:MapReduce

大数据计算模式

  • 批处理计算
  • 流计算
  • 图计算
  • 查询分析计算

没种技术都有其局限性，因此不同场景下会使用到不同的技术。

批处理计算

针对大规模数据的批量处理

常见的批处理计算框架:

  • MapReduce：分钟级响应
  • Spark: 1.数据放置在内存中进行计算 2.接近准实时性(秒级响应)

流计算

针对流数据的实时计算(常用场景有大型应用系统的故障分析检测)

常见的流计算框架:

  • Storm:毫秒级响应
  • S4
  • Flume
  • DStream

图计算

常见场景有社交网络系统

常见的流计算框架:

Pregel

Hama

Power Graph

查询分析计算

在数据仓库中使用 SQL 语句来进行查询分析

常见的查询分析计算框架:

  • Dremel
  • Hadoop 生态圈的 Hive
