使用Spring Boot创建定时任务

Mar 15, 2019 | 开发经验 | 阅读

实际业务代码实现中，有时候会需要定时任务来完成一些功能。Spring Boot 通过 @Scheduled 注解可以非常方便地实现此类功能。

环境

​ 首先需要知晓 @Scheduled 的环境要求。如图，在 Spring 官网上，我们可以知道它主要有两点环境要求。一是，需要 JDK 1.8 以上的 Java 环境；二是，需要 Gradle 4 以上或者是 Maven 3.2 以上。

因为我的项目是 Maven 项目，所以自然选择了 Maven 进行配置。

pom.xml

配置使用

创建一个定时任务类 ScheduledTasks

@Scheduled 注解可以有三种定义。 如上面示例使用的是 fixedRate ，表示的是每间隔一段时间执行该任务,无论上一次任务是否完成，单位为毫秒。还有其他选择，如 fixedDelay ，表示的是上一次任务完成之后，再间隔一段时间执行该任务。还可以使用 @Scheduled(cron=". . .") 来进行更为复杂的操作。

cron 表达式的使用示例:

  • "0 0 * * * *" = 每天每小时的0分0秒
  • "*/10 * * * * *" = 每十秒钟
  • "0 0 8-10 * * *" = 每天的8点整，9点整，10点整
  • "0 0 6,19 * * *" = 每天的上午6点整与下午7点整
  • "0 0/30 8-10 * * *" = 每天的8:00, 8:30, 9:00, 9:30, 10:00, 10:30
  • "0 0 9-17 * * MON-FRI" = 周一到周五的9:00 am-5:00 pm 的每一个整点
  • "0 0 0 25 12 ?" = 每个12月 25 日的 零点

该模式是六个由空格分隔的字段组成:分别表示秒，分钟，小时，日，月，工作日。

最后一步是在启动类上加上 @EnableScheduling 注解，这个注解确保创建了后台任务执行器，不加该注解定时任务将不会执行
